# 项目路线图 (Roadmap)

本文档概述了 TYPE 项目的未来发展方向和里程碑。我们欢迎社区成员参与讨论和贡献，共同推动项目成长。

---

## 🎯 愿景

让 TYPE 成为程序转换领域最灵活、易用的开源框架，支持从教学实验到工业应用的广泛场景。

---

## 📅 版本规划

### v0.1.x - 基础稳定版（当前）
- [x] 九种核心翻译语义的初步实现
- [x] 引入插件系统，允许用户用 Racket 编写自定义语义扩展
- [x] 多阶段流水线支持
- [x] 基础规则加载与解析
- [x] 示例规则与文档

**目标**：验证核心架构，收集早期用户反馈。

---

### v0.2 - 增强功能与可用性（预计 Q3 2026）

#### 功能增强
- [ ] 完善所有语义的实现
- [ ] 支持 ELF 包装器
- [ ] 提供类 C 语言源程序标准规则库
- [ ] 提供 risc-v 标准规则库

#### 文档与测试
- [ ] 完整的 API 文档
- [ ] 规则编写教程（从入门到进阶）
- [ ] 测试覆盖率达到 80%

---

### v0.3 - 性能与扩展性（预计 Q1 2027）

#### 核心优化
- [ ] 翻译引擎性能优化：支持大程序（>10 万节点）快速转换
- [ ] 并行翻译阶段支持（多线程处理独立阶段）
- [ ] 依据现代编译原理进行的算法优化
- [ ] 内存占用优化，减少中间文件生成

#### 新特性
- [ ] 支持非 S 表达式输入，通过适配器转换
- [ ] 图形化规则编辑器（可选 Web 界面）

---

### v1.0 - 稳定发布（预计 Q3 2027）

#### 发布条件
- [ ] 核心 API 稳定，无重大变更计划
- [ ] 文档完整，包含中英文版本
- [ ] 至少 3 个非示例的实际项目使用 TYPE 进行程序转换
- [ ] 所有已知严重 Bug 修复
- [ ] 长期支持承诺（LTS）

---

## 💡 如何参与路线图制定

路线图是社区共同制定的，我们欢迎任何建议和讨论。您可以通过以下方式参与：

1. **在 GitHub Discussions 中发起讨论**：提出新想法或对现有里程碑的建议。
2. **参与规划会议**：项目维护者会定期举行线上会议，欢迎贡献者加入。
3. **认领任务**：查看 [GitHub Issues](https://github.com/fanguangping/type/issues) 中带有 `help wanted` 或 `good first issue` 标签的任务，从小的改进开始。

---
